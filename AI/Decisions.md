# Decisions & Rationale — SAKIGo

The *why* behind design choices, so later sessions don't relitigate settled ground or lose the reasoning. Newest first. Entries are living until built, then they freeze.

## D39 - Budget uses raw visits and expands symmetry orbits (2026-07-14)

The active budget target uses KataGo's raw visit field `v`, not adjusted visits `av`. When an exported representative row lists multiple symmetry-equivalent coordinates, every listed concrete action receives the row's full visit count before the action vector is normalized. Target eligibility likewise requires positive raw concrete visits. The current full-capacity run reserves 1,024 validation records in each of six board-size/ruleset cohorts and allocates the remaining 19,117,495 fully labeled records to training. Direct-book record schema advances from v2 to v3 to distinguish these semantics; readers retain v1/v2 compatibility, and sample-allocation identity advances independently to invalidate stale frozen outputs. **Why:** `v` is the actual search-visit statistic requested for the budget head, and a symmetry row represents one visit count for every equivalent concrete action rather than mass that should be divided among them. The corrected eligibility scan recovers the older 7x7 and 8x8 books, which export `v` but not `av`. This supersedes D34-D35's visit-field semantics and their original sample totals while retaining their direct-book/no-ownership scope. Source: user direction, KataGo book export semantics, and the six preserved book-index audit on 2026-07-14.

## D38 - Validation is a stable board-size/ruleset matrix (2026-07-14)

The current large run allocates 6,144 validation records evenly across Tromp-Taylor and Japanese at each of 7x7, 8x8, and 9x9: exactly 1,024 per cohort. Training remains a global population-weighted draw from the eligible nodes left after validation is reserved. Prepared manifests record cohort counts; validation batches never cross board-size or ruleset boundaries, and every metrics row covers every cohort selected by the validation batch budget. Runs retain aggregate `metrics.csv` while adding long-form `validation_metrics.csv` and TensorBoard `val_groups/` curves for total and per-head loss. **Why:** one aggregate validation number can hide a regression in a board size or ruleset, while stable cohort curves show whether each part of the intended domain is learning. Source: user direction and implementation on 2026-07-14; total updated by D39.

## D37 - Resume and ingestion bind to control and content identity (2026-07-13)

Current checkpoints bind exact continuation to trajectory-defining run properties, the prepared-data content identity, optimizer/scheduler state, sampler queues, augmentation RNG, and global RNG; both checkpoint creation and resume require `num_workers=0` for exact control state. This guarantees the same batch/control sequence, not bit-identical floating weights, because backend kernels can be numerically non-bit-reproducible. The size sampler also records its version, batch size, pool counts, weighting mode, and weights, and prevents a pool wrap from repeating an index inside one batch. Generated shards and preparation sources carry SHA-256 identities; reuse, train/validation separation, and both preparation passes fail closed on mismatch. Pre-current-schema checkpoint migration remains intentionally out of scope. **Why:** silent changes to data, batching, or optimization invalidate an experiment more seriously than a loud restart, while claiming bit-exact arithmetic would overstate what the backend guarantees. Source: user dispositions and implementation audit on 2026-07-13.

## D36 - Match the board-attention work proxy across the depth/width sweep (2026-07-13)

The active comparison uses power-of-two bottlenecks with reciprocal depths: `narrow-deep` is `n=32, L=32`, `balanced` is `n=64, L=16`, and `wide-shallow` is `n=128, L=8`. All three hold `L*n=1024`, the leading board-token attention-mixing proxy, exactly fixed. Trunk parameter counts are deliberately not matched: 3,836,706; 5,405,426; and 8,549,082 respectively. **Why:** the sweep is intended to compare how depth and per-step width spend a fixed attention-mixing budget; forcing equal parameter counts produced irregular widths/depths and changed that compute proxy. This supersedes D31's parameter-matched points while retaining its fixed D4 architecture and block program. Source: user direction, `ModelSpecs.json`, and `ModelArchitecture.md`.

## D35 - Mixed 7x7/8x8/9x9 book distillation dataset (2026-07-13; sample totals superseded by D39)

The active dataset contains `2^20` training and `2^12` validation positions from six KataGo books: Tromp-Taylor and Japanese books at each of 7x7, 8x8, and 9x9. D38 supersedes this entry's original fully global validation allocation: training remains population-weighted with no fixed size or ruleset quota, while validation is cohort-balanced. Training batches remain single-size for tensor compatibility, while rulesets mix naturally and board-size batch tickets follow available record counts. Teacher inference, continuations, ownership, and a test split remain absent. **Why:** mixing all available small-board books broadens board-size and rule coverage without changing the deterministic book-only target construction. This supersedes D34 only for dataset composition; D34's no-inference/no-ownership target semantics remain active. Source: user direction and `sakigo.generate.multi_book_distillation.py`.

## D34 - Book-only 9x9 dataset with no teacher inference or ownership (2026-07-13; composition superseded by D35, visit semantics superseded by D39)

The active dataset contains exactly `2^20` training positions and `2^12` validation positions sampled uniformly by stable hash from validated canonical nodes in the 2026-02-26 TT7 book. Pages containing an aggregate `other` row remain eligible, while that row is discarded and concrete `AVisits` are renormalized for the budget target. Policy, score, and WDL are derived directly from concrete book rows. Teacher-model queries, continuation generation, late-game coverage, ownership targets and output head, inference-batch tuning, and a test split are dropped; checkpoint schema advances to 6 for the removed head. A 100,000-page audit found no page lacking a concrete move, `ssM`, `wl`, or positive concrete `av`, so inference fallbacks are unnecessary. Training-record schema v2 differs from v1 only by permitting policy mass uniform across concrete book moves tied after rounding to 0.5; v1 remains readable and strictly one-hot. **Why:** the book already filters toward sensible positions, and removing inference turns a potentially multi-day generation job into deterministic archive indexing and local replay. This supersedes D21 and D22 for the current dataset. Source: user direction, downloaded book audit, and `sakigo/generate/book_distillation.py`.

## D33 - Run the scalar board stem before the D4 lift (2026-07-12)

The board stem now applies scalar pointwise maps `6 -> 16 -> 128` with SiLU and lifts the completed 128-channel board into the D4 regular representation only once. On a lifted scalar input, every regular output fiber from the former stem was identical and depended only on the sum of each regular layer's eight group-component weights. The scalar-before-lift form therefore preserves the stem's effective function class and whole-model equivariance while reducing stem parameters from 17,296 to 2,288 and dense stem MACs by 64x. Trunk widths, parameter counts, and the depth/width comparison are unchanged. Checkpoint schema advances from 4 to 5 because the stem parameter shapes and optimizer slots change. **Why:** orientation-sensitive processing begins only in the regular trunk, so paying regular-representation cost before that point is redundant. This supersedes D13's original early-lift stem placement while retaining its end-to-end D4 requirement. Source: user direction, equivalence derivation, and implementation on 2026-07-12; [Stem.md](../Design/Architecture/Stem.md).

## D32 - Fuse attention projections without changing the architecture (2026-07-12)

Each board self-attention now evaluates its independent Q/K/V parameter slices with one fused regular QKV projection. Register gather and broadcast retain a separate query projection but evaluate their independent K/V slices with one fused regular KV projection, because K and V always share the same source stream. Gather uses `Q=registers, K/V=board` and updates only the register residual; broadcast uses `Q=board, K/V=registers` and updates only the board residual. Fusion preserves the exact attention equations, widths, D4 tying, parameter counts, and two-attention block prior while reducing projection launches and repeated input reads. Checkpoint schema is bumped from 3 to 4 because model parameter keys and optimizer parameter slots change even though their concatenated values are equivalent. **Why:** this is a low-risk execution optimization with a direct linear-map equivalence and a smaller launch surface. This is a current correction to D24's historical state-dict-shape preservation claim. Source: user direction, implementation, and equivalence/equivariance tests on 2026-07-12.

## D31 - Fixed D4 trunk and parameter-matched depth/width sweep (2026-07-12; sweep points superseded by D36)

The active model is D4-only and has one fixed program: rule features directly initialize two 64-wide registers; one 32-wide register attention broadcasts into a 128-wide board stream; `L` plain board blocks each apply `128 -> n`, two independent self-attention updates, and `n -> 128`; one final gather refreshes the registers for global and pass heads. The two-attention block is the fixed expressivity prior for the current sweep rather than another ablation axis. FiLM, SwiGLU, scalar control, repeated register cycles, and general operation-layout experiments are dropped. The packaged sweep controls the trunk at a target of 5,405,426 parameters with 0.2% tolerance: `narrow-deep` uses `n=40, L=33` (5,407,669), default `balanced` uses `n=64, L=16` (5,405,426), and `wide-shallow` uses `n=128, L=5` (5,398,737). With fixed `m=128`, one block contains `48*n^2 + 2058*n + 259` parameters and the broadcast plus gather add 148,162, so `P_trunk = 148,162 + L*(48*n^2 + 2058*n + 259)`. Equal parameters do not imply equal runtime: attention mixing scales approximately with `L*n`, while sequential depth and retained activations also grow with `L`. Training uses `torch.compile(mode="reduce-overhead")` by default based on the local fixed-shape BF16 benchmark. **Why:** fixing symmetry, persistent width, register path, block structure, and parameter budget isolates the useful architectural question—narrow/deep versus wide/shallow—without carrying unrelated choices. This supersedes D28's general-layout direction and D29's FiLM architecture for current work; both entries remain below as history. Source: user direction and parameter audit on 2026-07-12; [ModelArchitecture.md](../Design/Architecture/ModelArchitecture.md).

## D30 - External one-visit KataGo distillation scope (2026-07-12; superseded by D34-D35)

Current work covers one-visit KataGo teacher-data generation, student training, and distilled-checkpoint evaluation. Search, self-play training, subtree harvesting, best-move visit cutoffs, branched games, feature/time auxiliary heads, percentile score heads, and high-visit Phase 2 are not currently considered. Budget remains an active head only as the soft KataGo-policy target; policy remains the teacher top-1 target. Earlier search/self-play decisions are retained as history but do not direct current implementation or planning. **Why:** the project benefits more from completing and validating the smallest coherent distillation pipeline than from carrying speculative future systems in the active scope. Source: user direction on 2026-07-12.

## D29 - Registers FiLM only the bottleneck delta (2026-07-12; superseded by D31)

Default conditioned blocks derive invariant per-channel scale and bias from the register stream and apply them to the normalized `n`-wide bottleneck result immediately before `f4`. The persistent `m`-wide residual stream is never directly modulated. Register fibers are RMS-normalized, averaged over the group axis, flattened across register tokens, and passed through `R*r -> register_bottleneck_channel -> 2*n` with SiLU; the final projection is zero-initialized so the path begins as identity. FiLM is instantiated only by explicit `film_block` operations. The default uses gather-first cycles at blocks 1, 6, 11, and 16 (`gather -> broadcast -> film_block`) plus a final gather for global heads; ordinary blocks own no FiLM parameters. **Why:** gather makes the register context current, then broadcast and FiLM use that same context for complementary spatial and channel conditioning without touching persistent memory directly. Source: user discussion and implementation on 2026-07-12; [ModelArchitecture.md](../Design/Architecture/ModelArchitecture.md).

## D28 - The trunk is an ordered program of independent modules (2026-07-12; superseded by D31)

Board blocks, FiLM board blocks, register gathers, and register broadcasts are independent residual modules assembled in the order declared by the model spec. Gather is no longer forced to read a board block's input, and broadcast is no longer forced to write its output. The default `register_cycles_1_6_11_16` layout uses gather-first register cycles and a final gather. Model-spec schema 4 uses shared `model_defaults`, named `trunk_layouts`, and small per-model overrides instead of repeating the complete architecture for every experiment. **Why:** register timing is an architectural choice, not an intrinsic part of a board block; an ordered program expresses arbitrary schedules directly, while defaults and overrides keep model experiments maintainable. Checkpoint compatibility with the former fused blocks was intentionally not preserved during the move-quick phase. Source: user discussion and implementation on 2026-07-12; [ModelArchitecture.md](../Design/Architecture/ModelArchitecture.md), [ModelSpecs.json](../sakigo/model/specs/ModelSpecs.json).

## D27 - Phase-1 boundaries fail closed and resume exactly (2026-07-11; historical, formats advanced by D34-D35 and current contracts)

Generation now forces KataGo analysis values to Black perspective, owns and reaps the subprocess across every exit path, records failed status, times out stalled responses, and refuses to reuse output shards unless overwrite is explicit. Schema-v1 loading is strict about ruleset/version, boolean legality, pass legality, one-hot policy, and zero illegal target mass. Prepared-data format v2 splits on canonical model-visible input bytes and writes each rebuild into an immutable generation directory before atomically switching the manifest. Checkpoint schema v3 captures sampler and augmentation state; exact resume uses `num_workers=0`, and non-exact prefetched states are rejected. Evaluation safe-loads checkpoints by default, represents draws and capped games honestly, uses paired uncertainty for paired self-play, and rejects Ancient Chinese scoring until canonical engine scoring exists. `StateHash` now includes exact komi bits and a digest of superko history. **Why:** experiment conclusions are only as trustworthy as their data, split, resume, and adjudication boundaries; silent fallback or approximation is worse than a loud refusal. Source: architecture/implementation review and hardening implementation on 2026-07-11.

## D26 - Rule MLP directly initializes registers (2026-07-07)

`SakiGoNet` no longer has a separate learned `register_seed`. Initial registers are now just `reshape(rule_mlp(rules))` expanded across the D4 group axis. **Why:** the final bias of `rule_mlp` can already learn a rule-independent default register state, so `register_seed + rule_mlp(rules)` was redundant while making the architecture harder to reason about. Old checkpoint states that still contain `register_seed` are tolerated by dropping that obsolete key during load. Source: user discussion and implementation on 2026-07-07.

## D25 - Register stream can be narrower than the board trunk (2026-07-07)

Model specs split board/trunk width from register width: `expanded_channel` is the spatial trunk width, while `register_channel` controls per-register features and `register_bottleneck_channel` controls register gather/broadcast attention width. Cross-attention uses rectangular projections, so registers can query/broadcast with different input/output widths. At the time, `plain` and `swiglu` used 128-wide board trunks with 64-wide registers and 32-wide register cross-attention, while `non-bottleneck` used a 64-wide board trunk. D31 retains the 64/32 register widths but fixes every board trunk at 128 and replaces those experiments with the depth/width sweep. **Why:** narrower registers keep the global readout path while avoiding the cost of treating registers as a full-width copy of the spatial stream. Source: user discussion and implementation on 2026-07-07; D31.

## D24 - Equivariant attention is a reusable finite-group library (2026-07-07)

The regular-representation attention machinery has been generalized and decoupled into `equivariant_attention/`: `FiniteGroupSpec` plus trivial/Cn/D4 square-grid presets, regular linear/norm/MLP layers, invariant pooling, spatial self-attention, and spatial/register cross-attention. `sakigo.model.group` and `sakigo.model.layers` now act as SAKIGo-compatible wrappers around that package, preserving old class names and state-dict shapes. **Why:** the useful object is not Go-specific; it is finite-group equivariant attention with canonical-frame position encoding, and keeping it as a small torch-only library makes it reusable across future board/grid projects without copying the algebra and attention code. Source: user request and implementation on 2026-07-07.

## D23 - Large training data uses zstd JSONL shards and a DataLoader streaming buffer (2026-07-03, updated 2026-07-05; superseded by prepared mmap data)

`Training.train` now defaults to bounded streaming (`--stream-buffer-mb 1024`) through PyTorch `IterableDataset` + `DataLoader` wrappers. The preferred data layout is numbered `.jsonl.zst` shards such as `samples_000000.jsonl.zst`; plain single `.jsonl` remains readable but is marked deprecated, and eager loading is only available with `--stream-buffer-mb 0`. Streaming scans metadata once, splits positions deterministically by hash, keeps decoded records under the requested buffer budget, and collates homogeneous/ruleset-balanced batches through DataLoader. **Why:** the 2^18 Phase 1 file is too large as expanded Python objects, while zstd shards avoid bloated individual files and make data management cleaner. Historical sources (removed at cutover): `Training/train.py`, `Training/data.py`, `Training/generate_katago_phase1.py`.

## D22 - Local KataGo Phase 1 batch-inference setting (2026-07-03, empirical; legacy-only under D34-D35)

For local Phase 1 KataGo teacher generation on the RTX 5070 Ti Laptop GPU with the bundled TensorRT KataGo v1.16.5 and `kata1-zhizi-b40c768nbt-s11272M-d5935M`, use `maxVisits: 1`, `analysisPVLen: 1`, `includePolicy: true`, `includeOwnership: true`, `includeNoResultValue: true`, `numSearchThreadsPerAnalysisThread=1`, `nnMaxBatchSize=20`, and `numAnalysisThreads=40`. At the generator level, think "GPU batches of 20 NN evaluations" but keep about 40 positions/games in flight so KataGo actually fills those batches. **Why:** a steady-state benchmark on 4096-8192 unique four-move 19x19 positions found batch 20 / 40 analysis threads at about 226-227 samples/s, narrowly ahead of batch 16 / 32 threads and clearly ahead of 8, 12, 24, 32, 64, and 128. Bigger TensorRT batches filled well but were slower for this model/hardware. Source: local benchmark 2026-07-03.

## D21 - Phase 1 KataGo labels map budget to raw policy and policy to top-1 (2026-07-03; superseded by D34-D35)

Phase 1 uses the external KataGo teacher without search/with one visit, trains budget on KataGo's raw policy output, and trains policy on the top-1 move derived from that output. WDL maps from raw win/draw/loss/no-result values, score from raw lead, and ownership from the root ownership map, with mover-perspective conversion. No current search-time role is assigned to either action head. **Why:** this is the smallest direct teacher-to-student target mapping supported by the generator. ## D20 - Engine owns legality, history, and neural encoding (2026-07-03, implemented)

The Rust engine is the deterministic source of truth for board topology, captures, suicide, simple ko, positional superko, pass moves, capture accounting, history hashes, and model input encoding. ## D18 - Model v1 uses rule-conditioned D4 attention with no FiLM branch (2026-07-03, updated 2026-07-07)

The implemented `SakiGoNet` applies a scalar pointwise board stem, lifts its output to regular features, initializes equivariant register tokens directly from `rule_mlp(rules)`, executes regular spatial GQA board blocks and register exchanges, and collapses the D4 axis only in heads. The original sequence broadcast rules before board reasoning and gathered the final board representation; D29 temporarily added invariant register-derived FiLM to selected bottleneck deltas. **Current corrections (D31, D33):** the fixed active sequence is one broadcast, only plain two-attention board blocks, and one final gather, with no FiLM; the D4 lift occurs after the scalar stem. **Why:** this turns the SquareAccumulation reference and current design docs into a concrete, tested baseline while keeping rule conditioning structurally contained. Source: [model.py](../sakigo/model/model.py), [layers.py](../sakigo/model/layers.py), [EquivariantAttention.md](../Design/Architecture/EquivariantAttention.md), [ModelArchitecture.md](../Design/Architecture/ModelArchitecture.md).

## D17 — Policy can be sharp; budget should stay smooth (2026-06-17, clarified; superseded by D34-D35)

In current distillation, budget matches the teacher's smooth raw policy distribution and policy matches its derived top-1 move. No search-time interpretation is currently assumed. **Caveat:** the hard policy target is derived from the same teacher distribution, so it can duplicate action supervision and amplify teacher noise; keep its loss weight explicit. Source: human clarification and D30.

## D16 — Search direction: Bayesian value-of-information leaf selection (inactive under D30)

Leaning toward a Bayesian searcher: expand the leaf with the highest probability of changing the root policy, rather than a PUCT bonus. **Why:** it's the rational-metareasoning ideal (value of information / value of computation; Russell–Wefald, Tolpin–Shimony VOI-MCTS, Tesauro et al. Bayesian MCTS) that UCT/PUCT only approximate — mathematically the cleanest statement of "spend the next sim where it matters." **Tensions to resolve before committing** (see [Issues.md](Issues.md)): (1) VOI is decision-focused → concentrates sims on the top-2 moves → degenerate *training* target (the conflict KataGo patches with forced playouts) — clashes with D5/D10. (2) "Change the policy" is ambiguous: root *argmax* (→ binary VOI, top-2) vs root *visit distribution* (→ a KL magnitude, not a probability). (3) Bayesian cleanness needs posteriors propagated through max nodes (not closed-form) + myopia — approximations re-enter. **Pragmatic realization:** Gumbel/Sequential-Halving for the decision-focused behavior without posteriors, and reframe the **budget head as a learned VOI** (regress realized policy-change-per-visit) — turns the intractable analytic VOI into a training target and gives the budget head (D12) a crisp definition. Source: author, this session.

## D13 — Stem: small group-equivariant CNN, regular representation (2026-06-17; equivariance scope corrected 2026-06-18)

A small G-CNN (regular rep) as the stem. **Why:** bakes in Go's D4 board symmetry structurally instead of relying only on data augmentation. **Whole-net equivariance is the goal** (corrected 2026-06-18, clarified 2026-07-03): stem, D4-equivariant spatial attention trunk (D14), and register-token attention are all meant to be equivariant — not a stem-only front-end prior. **Open:** regular rep multiplies channels by |G| (×8 for D4) — costly, hence "small"; confirm that beats plain symmetry augmentation. The hard part is keeping spatial attention and register-token QKV read/write equivariance-preserving, since arbitrary QKV/channel mixing is not equivariant (see [Issues.md](Issues.md)). Color-swap / komi / Boundary on non-square boards aren't spatial symmetries and don't apply. Source: [Stem.md](../Design/Architecture/Stem.md); human clarification 2026-06-18 and 2026-07-03.

## D14 — Trunk: equivariant spatial attention + register-token attention (2026-06-17; clarified 2026-07-03)

The trunk uses D4-equivariant spatial attention nested residual blocks plus register-token QKV attention. Registers **read and write** spatial tokens (a bidirectional global-context exchange, not a read-only summary), and both spatial attention and register attention preserve whole-net equivariance (D13). **Why:** spatial attention provides board reasoning directly, while registers provide a compact global pathway and become the native input for global heads (D15). Rule conditioning initializes register tokens by default. A working reference implementation came from the owner's playground repo (`D:\stuff\Documents\SquareAccumulationK-Isolation`) — regular-rep fibers, group-axis-batched QKV, canonical-frame RoPE, equivariance-tested, no escnn. **D31 closes the remaining schedule/conditioning choice:** one broadcast, plain board blocks only, and one gather. Source: [Trunk.md](../Design/Architecture/Trunk.md); human clarification 2026-06-18 and 2026-07-03; D31.

## D15 — Heads: 1×1 conv (spatial) + register MLP (global) (2026-06-17; clarified 2026-07-03)

Spatial heads = 1×1 convs; global heads = MLPs applied to register tokens. **Why:** matches the spatial/global split ([SpatialGlobalDistinction.md](../Design/Output/SpatialGlobalDistinction.md)); the register stream is already the global state, so a separate attention-pooling mechanism is unnecessary unless later evidence asks for it. Source: [Heads.md](../Design/Architecture/Heads.md).

## D12 — Harvest cutoff keyed on best-move visits, not total visits (inactive under D30)

Gate harvest on the most-visited child's count (best-move visits ≥ K), not node-total N or a flat playout cap. **Why:** wide/uncertain positions split visits, so they must accumulate more total search before the top move reaches K — compute flows to noisy positions, and a node is harvested only once its principal variation is trustworthy (a flat-N gate can pass a wide node whose best move is still under-visited). **Open:** is this a *dynamic search-termination* rule (search until best move hits K → genuinely allocates more compute) or a *post-hoc filter* on fixed-budget trees (then it *rejects* high-entropy nodes — the opposite effect)? Near-ties need a total-visit ceiling. Source: [BestMoveVisit.md](../Design/Train/BestMoveVisit.md).

## D10 — Training is search-based self-play, distilling search into the prior (inactive under D30)

The net (student) learns to approximate the result and statistics of net+search (teacher). **Why:** search is the policy-improvement operator; distilling it back into the prior is the AlphaZero/KataGo self-improvement loop. "Statistics" = visit distribution → budget/policy prior; "result" = root value → winrate/score. **Clarification (2026-06-19, human):** this is the *self-play RL loop* — the "teacher" is the net's **own search over its self-play games**, with targets outcome-grounded by the game result z. It is **not** offline distillation from a fixed external teacher (the vibego setting); "distillation" here means distilling *search into the prior*, not learning from a separate stronger net. This re-scopes the vibego evidence in [Issues.md](Issues.md) — most of its distillation-specific cautions do not transfer. **Current tension (2026-07-03):** [Target.md](../Design/Distillation/Target.md), `Distillation/` assets, and `Model/sakigo_model/adapters.py` now sketch a separate KataGo-teacher distillation/bootstrap path. Treat that as unresolved against this older self-play-only framing until the training plan is reconciled. Source: [SearchBasedStudentTeacher.md](../Design/Train/SearchBasedStudentTeacher.md), [Target.md](../Design/Distillation/Target.md).

## D11 — Subtree harvest: train interior tree nodes, not just the root (inactive under D30)

Root search with n visits yields f^n(x_t) as the root's target; each interior node x_t^p already accumulated m visits, so train f(x_t^p) → f^m(x_t^p) too instead of discarding the subtree. **Why:** the subtree NN evals are already paid for, so this adds training targets at zero extra search cost. **Two gates make it sound:** (1) a **min-visit cutoff** — only harvest nodes searched enough that f^m is a real refinement; directly analogous to KataGo's playout-cap randomization, where only full-playout moves supply policy targets. (2) **policy/budget-entropy gating** — harvest only high-entropy nodes, which excludes peaked/forced near-duplicate runs (kills intra-tree correlation) and de-skews away from confidently-greedy lines (kills most selection bias). **Still open:** harvest is search-bootstrapped, never outcome-grounded (no z at interior nodes), and both gates key off the *current* PUCT, which is itself under redesign — see [Issues.md](Issues.md). **Reaffirmed for self-play (2026-06-19, human):** author considers subtree harvest well-suited to self-play training — the interior searched values *are* the policy-improvement signal, and the self-play outcome z anchoring the played line partially mitigates the "no z at interior nodes" concern (a watch-item, not a blocker). Source: [SubTreeHarvest.md](../Design/Train/SubTreeHarvest.md).

## D9 — History / last-move prior intentionally dropped (2026-06-17)

No history planes. Go is only fully Markovian given complete history, but the history-aware engine computes ko/superko legality and projects it into the `NonTrivialIllegal` plane for every distillation record. The teacher rollout still retains full game history; only the student input is intentionally lossy. History's remaining last-move-response prior is deliberately omitted to keep the input small and avoid that bias. Source: human clarification and [Markov.md](../Design/Input/Markov.md).

> **Seed (2026-06-17):** D1–D8 below were **captured from the existing `../Design/` notes**, not decided by the AI. They record the human's choices and stated rationale so the reasoning survives across sessions. (They share one date, so they read D1→D8; future entries go on top, newest first.)

## D1 — Minimal board input: 6 planes

MyStones, OpponentStones, EmptyPositions, BoundaryCorner, BoundaryEdge, NonTrivialIllegal. **Why:** "leave it to the model to figure it out" — avoid hand-engineered features while making occupancy, board geometry, and non-trivial legality explicit. Boundary planes also enable non-rectangular boards. Source: [BoardInput.md](../Design/Input/BoardInput.md); human clarification 2026-07-03.

## D2 — Rules directly initialize registers

Rule settings are one-hot encoded and concatenated with scalar komi/capture-difference inputs. This non-board vector passes through an MLP to initialize the register tokens; one initial broadcast is the only rule-to-board conditioning path. FiLM is not part of the active architecture under D31. **Why:** feeding every rule as a board plane is overhead and rarely sampled; one-hot keeps correlated rules from being double-counted, and registers are the native global state read by global heads. Source: [NonBoardInput.md](../Design/Input/NonBoardInput.md); human clarification 2026-07-03 and D31.

## D3 — Only a subset of rules, one-hot per correlated group

Scoring {Area, Area+AncientChinese, Territory, TerritoryWithSekiScore}, Ko {SimpleKo, PositionalSuperKo}, Suicide {Yes, No}. **Why:** Go's rules don't map cleanly onto a hypercube — some rules exclude others, so a full cross-product would waste capacity and rarely be sampled. Source: [NonBoardInput.md](../Design/Input/NonBoardInput.md).

## D4 — Komi & captured stones as normalized scalars

Two *retained* scalars in [−1, 1], divided by board **area** (matching the score head, D7; handicap folded into komi implicitly). CapturedStones means `#opponent stones I captured - #my stones opponent captured`, also normalized by board area. **Why:** compact and board-size-agnostic; the prisoner count is needed for Territory scoring, so it stays. Source: [NonBoardInput.md](../Design/Input/NonBoardInput.md); human clarification 2026-06-18 and 2026-07-03.

## D5 — Policy and budget are separate targets (mapping superseded by D34-D35)

The two action heads remain separate, and pass is one additional logit beside the n² board moves in each softmax. The original teacher-policy/top-1 mapping was superseded by the direct-book targets in D34-D35 and corrected by D39: normalized raw concrete visits (`v`), expanded across listed symmetry-equivalent actions, for budget and uniform tied rounded-optimum moves for policy. Source: [Policy+Budget.md](../Design/Output/Policy+Budget.md); D34-D35; D39.

## D6 - Win / Draw / Loss / No-result as a length-4 vector

The WDL head is length 4: win, draw, loss, and no-result. **Why:** draws and no-result are distinct outcomes, so winrate cannot collapse to a single scalar. Source: [Winrate.md](../Design/Output/Winrate.md), [HeadShapes.json](../sakigo/model/specs/HeadShapes.json).

## D7 — Score: normalized scalar only in the current scope

The current distillation target is scalar score normalized by board area. Percentile and other distributional score heads are inactive under D30. Source: [Score.md](../Design/Output/Score.md).

## D8 — Auxiliary heads as temporal predictions (inactive under D30)

This historical proposal predicted a main head's value n plies ahead. Feature and time auxiliary heads are not currently considered. Source: [TimeAuxiliary.md](../Design/Output/TimeAuxiliary.md); D30.
